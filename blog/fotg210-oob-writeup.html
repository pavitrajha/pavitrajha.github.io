3x slab-out-of-bounds in fotg210 UDC via attacker-controlled wIndex

Bounds-check omission in drivers/usb/fotg210/fotg210-udc.c leads to three independently reachable out-of-bounds accesses of fotg210_udc.ep[].

The endpoint array is declared with FOTG210_MAX_NUM_EP = 5 (valid indices 0–4), but request handlers derive an endpoint number from: 

ctrl->wIndex & USB_ENDPOINT_NUMBER_MASK

This produces values 0–15 with no bounds validation before indexing fotg210->ep[epnum]. A malicious USB host can trigger slab OOB reads beyond the fotg210_udc allocation.

1. Environment

Kernel    : Linux 7.0.0-rc7+
Arch      : x86_64 (QEMU i440FX)
KASAN     : enabled (generic + inline)
KASLR     : disabled (nokaslr)

2. Threat Model

USB device controller runs in peripheral mode. Attacker is a malicious USB host. No authentication required. Attack surface is USB SETUP packet handling.

3. Root Cause

#define FOTG210_MAX_NUM_EP 5
struct fotg210_ep *ep[5];

But indexing uses: 

USB_ENDPOINT_NUMBER_MASK = 0x0f

Heap layout: 

ep[5] @ offset 952 → kmalloc-1k boundary at 1024
  • 0–4: valid
  • 5–8: intra-object corruption
  • 9–15: slab OOB

4. Vulnerability 1 — clear_feature OOB dereference

ep = fotg210->ep[ctrl->wIndex & USB_ENDPOINT_NUMBER_MASK];
ep->wedged

The pointer is immediately dereferenced without validation. 

BUG: KASAN: slab-out-of-bounds in fotg210_clear_feature+0x2ba/0x2d0
Read of size 8 at addr ffff888008d9ec30 by task insmod/61

fotg210_clear_feature+0x2ba/0x2d0
poc_init+0x1a7/0xff0

The buggy address is located 120 bytes to the right of allocated 952-byte region

Secondary crash:

KASAN: null-ptr-deref in range [0x0000000000000060-0x0000000000000067]
RIP: fotg210_clear_feature+0x158/0x2d0
RBP: 0x0000000000000000

7. Impact

  • Kernel slab out-of-bounds read
  • Cross-object heap interpretation
  • MMIO register manipulation via corrupted struct fields
  • Information leakage via USB response path

8. Fix

u8 epnum = ctrl->wIndex & USB_ENDPOINT_NUMBER_MASK;

if (epnum >= FOTG210_MAX_NUM_EP) {
    fotg210_request_error(fotg210);
    return;
}

struct fotg210_ep *ep = fotg210->ep[epnum];

All accesses must validate before dereferencing.
